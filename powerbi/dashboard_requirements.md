# Power BI Dashboard Requirements: PMO Project Portfolio Control System

## 1. Dashboard Objective

The objective of this Power BI dashboard is to provide executive visibility into the health of a project portfolio managed by a PMO.

The dashboard should help leadership answer the following questions:

- How many projects are active, completed, delayed, or over budget?
- Which projects require executive attention?
- Which departments have the highest risk exposure?
- Which project managers are handling the highest workload?
- Which projects have overdue milestones?
- Which projects have high-risk or escalated risks?
- How much budget is approved, spent, and forecasted?
- What is the overall traffic light status of the portfolio?

---

The dashboard is designed for:

- PMO Managers
- Project Managers

- Operations Leaders
- Department Managers
- Executive Leadership

## 4. Recommended Data Model

### 4.1 Main Table

The main table for executive reporting should be:

- project_portfolio_enriched

This table contains one row per project and includes project data, budget indicators, milestone summaries, risk summaries, and executive traffic light status.

### 4.2 Detail Tables

Use the detail tables when drill-down analysis is needed:

- milestone_tracker_clean
- risk_issue_log_clean
- budget_control_clean
- project_master_clean

### 4.3 Suggested Relationships

Relationship 1:

project_portfolio_enriched[project_id]  
1-to-many relationship with  
milestone_tracker_clean[project_id]

Relationship 2:

project_portfolio_enriched[project_id]  
1-to-many relationship with  
risk_issue_log_clean[project_id]

Relationship 3:

project_portfolio_enriched[project_id]  
1-to-1 relationship with  
budget_control_clean[project_id]

### 4.4 Relationship Direction

Recommended direction:

Single direction from project_portfolio_enriched to detail tables.

Reason:

The project portfolio table should act as the main project dimension for executive dashboard filtering.

## 13. Suggested DAX Measures

### Total Projects

Total Projects = COUNTROWS(project_portfolio_enriched)

### Active Projects

Active Projects =
CALCULATE(
    COUNTROWS(project_portfolio_enriched),
    project_portfolio_enriched[status] = "Active"
)

### Completed Projects

Completed Projects =
CALCULATE(
    COUNTROWS(project_portfolio_enriched),
    project_portfolio_enriched[status] = "Completed"
)

### Delayed Projects

Delayed Projects =
CALCULATE(
    COUNTROWS(project_portfolio_enriched),
    project_portfolio_enriched[is_delayed] = TRUE()
)

### Projects Over Budget

Projects Over Budget =
CALCULATE(
    COUNTROWS(project_portfolio_enriched),
    project_portfolio_enriched[is_over_budget] = TRUE()
)

### Projects Requiring Executive Attention

Projects Requiring Executive Attention =
CALCULATE(
    COUNTROWS(project_portfolio_enriched),
    project_portfolio_enriched[requires_executive_attention] = TRUE()
)

### Total Portfolio Budget

Total Portfolio Budget = SUM(project_portfolio_enriched[budget])

### Total Actual Cost

Total Actual Cost = SUM(project_portfolio_enriched[actual_cost])

### Total Forecast Cost

Total Forecast Cost = SUM(project_portfolio_enriched[forecast_cost])

### Total Budget Variance

Total Budget Variance = SUM(project_portfolio_enriched[budget_variance])

### Average Completion Percentage

Average Completion Percentage =
AVERAGE(project_portfolio_enriched[completion_percentage])

### Total Open Risks

Total Open Risks = SUM(project_portfolio_enriched[open_risks])

### Total High Risks

Total High Risks = SUM(project_portfolio_enriched[high_risks])

### Total Escalated Risks

Total Escalated Risks = SUM(project_portfolio_enriched[escalated_risks])

### Portfolio Risk Exposure

Portfolio Risk Exposure =
SUM(project_portfolio_enriched[portfolio_risk_exposure])

### Total Overdue Milestones

Total Overdue Milestones =
SUM(project_portfolio_enriched[overdue_milestones])

### Executive Attention Rate

Executive Attention Rate =
DIVIDE(
    [Projects Requiring Executive Attention],
    [Total Projects],
    0
)

### Over Budget Rate

Over Budget Rate =
DIVIDE(
    [Projects Over Budget],
    [Total Projects],
    0
)

### Delayed Project Rate

Delayed Project Rate =
DIVIDE(
    [Delayed Projects],
    [Total Projects],
    0
)

## 14. Recommended Color Logic

Traffic light colors should follow this logic:

- Green: Project is on track
- Yellow: Project requires monitoring
- Red: Project requires executive attention

Use conditional formatting in Power BI tables and charts based on:

- traffic_light_status
- risk_exposure_flag
- budget_variance_category

## 16. Dashboard Design Principles

The dashboard should follow these design principles:

- Executive summary first
- Exceptions and risks should be visible immediately
- Red/yellow/green logic should be consistent
- Avoid overcrowding pages
- Use slicers for department, project manager, priority, and status
- Use tables only when action or detail is needed
- Prioritize decision-making over decoration

---

## 17. Business Interpretation

A good Power BI dashboard should not only show numbers.

It should help leadership answer:

- What is going wrong?
- Where is it happening?
- Who owns it?
- How severe is it?
- What should be reviewed first?

This PMO dashboard should support project governance, portfolio visibility, budget control, and executive prioritization.
